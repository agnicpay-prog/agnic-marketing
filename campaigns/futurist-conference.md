# Campaign: Agentic Commerce Conference Evaluation (worked example)

Owner skill: [events](../skills/events/SKILL.md)

Priority tier: 3 (thought leadership — evaluate carefully, do not resource ahead of tier 1/2)

## Objective
Decide whether to speak/sponsor/attend a given "future of AI/agentic commerce" style conference, using the events filter.

## Evaluation (apply the [events](../skills/events/SKILL.md) skill's sponsorship filter before committing)

```
SPONSORSHIP: [conference name]
Cost: [$ + founder days]
Audience match: [how many real agent-wallet/x402/AI-payments builders or investors will attend?]
Alternative use of this budget: [e.g. 2 weeks of tier-1 outbound, or a partner recruiting sprint]
Decision: [yes/no]
Why: [one sentence]
```

## Default posture
Decline sponsorship spend. Accept a speaking slot only if: (a) it costs founder time, not meaningful cash, (b) the content can be repurposed into LinkedIn/X posts afterward (feeds the [community](../skills/community/SKILL.md) skill's tier-3 cadence), and (c) it doesn't pull founder time away from tier-1/2 execution in the same week.

## Core message/hook (if attending)
"Agentic commerce isn't a research topic — someone has to build the wallets, identity, and checkout rails agents actually use. We're doing that now."

## Assets needed (only if attending)
- Talk outline / talking points (founder talking points, see the [cmo](../skills/cmo/SKILL.md) skill's deliverables list)
- 3-5 post-conference LinkedIn/X posts distilling the talk (see [../templates/linkedin.md](../templates/linkedin.md))

## Owner
Founder decides attendance; Events Lead handles logistics only after a yes decision.

## KPIs
- Not revenue — track qualified conversations started (investors, future partners, press) and content produced from it.

## Risks
Easy to rationalize attending "for exposure." Treat this document as the check against that — if the filter says no, the answer is no regardless of how prestigious the event sounds.
